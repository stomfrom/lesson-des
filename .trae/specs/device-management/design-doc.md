# 星维设备管理系统 需求分析与设计文档

---

## 第1章 需求分析

### 1.1 范围

#### 1.1.1 编写的目的

本文档旨在明确星维设备管理系统的功能需求、系统架构和设计方案，为开发团队提供统一的开发依据，确保系统实现符合预期目标。本文档的预期读者包括项目开发人员、测试人员、项目管理人员及系统最终用户。

#### 1.1.2 项目背景

（1）系统名称：星维设备管理系统  
（2）用户：系统开发后，适用于企业内部设备管理人员，包括系统管理员和普通操作员两类用户。  
（3）开发者：实训项目开发小组  
（4）该系统独立运行，不依赖其他外部系统，通过 MySQL 存储设备、用户及权限数据，通过 RESTful API 为前端提供数据服务。

### 1.2 需求概述

#### 1.2.1 任务目标

构建一个企业级设备管理系统，实现对设备信息的全生命周期管理。系统需要支持设备信息的增删改查、状态跟踪、维保提醒，同时具备用户认证、权限控制和数据统计看板功能，提升设备管理效率与数据安全性。

#### 1.2.2 用户特点

系统面向两类用户：

**系统管理员**：负责系统的全面管理，包括用户管理和设备管理。具备较高的计算机操作水平，使用频率为每日多次，工作场景为企业办公环境。

**普通用户（操作员）**：负责设备的日常维护和操作。具备基础计算机操作能力，使用频率为每日定时使用，工作场景为车间或办公室。权限由管理员单独分配。

#### 1.2.3 运行环境

1、系统硬件要求

CPU 主频 2.0GHz 及以上，硬盘 10GB 以上可用空间，内存 4GB 及以上。

2、系统软件要求

| 类别 | 名称及版本 |
|------|-----------|
| 开发工具 | Visual Studio Code |
| 开发语言 | JavaScript (Node.js 18+), Vue 3 |
| 数据库管理工具 | MySQL Workbench |
| 数据库 | MySQL 8.0 |
| 操作系统 | Windows 10/11 或 Linux |
| 浏览器 | Chrome 90+, Firefox 90+, Edge 90+ |

### 1.3 功能需求

通过前期对设备管理领域实际业务需求的调研，经分析确定系统功能主要分为以下三个部分：

1. **设备管理**，包括设备信息增删改查、详情查看、CSV导出、维保提醒、状态筛选、表格排序。
2. **用户与权限管理**，包括用户注册登录、用户管理（管理员）、RBAC 权限分配。
3. **数据统计与系统配置**，包括 Dashboard 数据概览、设备生命周期自动流转配置。

#### 1.3.2 系统功能概述

设备管理功能支持设备信息的完整 CRUD 操作。用户可以查看设备列表（支持分页、按设备名模糊搜索、按状态多选筛选、列排序），点击行查看设备详情弹窗。新增和编辑设备时需填写设备名、型号、位置（必填），以及状态、维保日期、生命周期触发月数（可选）。删除设备时需二次确认并显示设备名。列表支持数据导出为 CSV 文件。

（2）用例图

设备管理用例图展示了用户对设备的全部操作。管理员拥有全部操作权限；普通用户仅拥有被管理员授权的操作（查看/新增/编辑/删除的任意组合）。

（3）用例文档

### 2.2 数据库设计

#### 2.2.1 概念结构设计

系统包含三个核心实体：用户（User）、设备（Device）、权限（Permission）。一个用户可以拥有多个权限记录，用户与权限为一对多关系。设备实体独立，不与用户直接关联，通过权限表控制不同用户对设备的操作权限。

```
┌─────────┐       ┌───────────────────┐
│  users  │──1──N→│   permissions     │
│ (用户表)  │       │  (权限表)          │
└─────────┘       └───────────────────┘
                        │
                        │ 控制对设备的操作
                        ▼
                 ┌──────────────┐
                 │   devices    │
                 │   (设备表)     │
                 └──────────────┘
```

#### 2.2.2 逻辑结构设计

用户信息（用户ID，用户名，密码，昵称，角色，创建时间，更新时间）
设备信息（设备ID，设备名，型号，位置，状态，上次维保日期，维保触发月数，报废触发月数，创建时间，更新时间）
权限信息（权限ID，用户ID，资源标识，操作标识，创建时间）

## 第3章 详细设计

### 3.1 功能模块详细设计

#### 3.1.1 设备管理功能

**1、功能描述**

设备管理用于系统中设备信息的全生命周期管理，支持设备信息的查询（分页+筛选+排序）、新增、编辑、删除、详情查看和 CSV 导出。表格所有列可点击排序，筛选条件变更自动重置分页，删除操作为二次确认并显示设备名。

**2、输入项目**

（1）用户查询设备列表：可输入设备名关键词、选择设备状态（多选）、切换页码和每页条数。
（2）用户新增设备：输入设备名、型号、位置（必填），以及状态、维保日期、维保触发月数、报废触发月数（可选）。
（3）用户编辑设备：修改上述任意字段。
（4）用户导出设备列表：点击导出按钮直接下载 CSV。

**3、输出项目**

（1）查询设备列表成功后，以分页表格展示设备数据，显示"共 N 条结果"。
（2）查询结果为空时，显示el-empty空状态组件"暂无设备数据"。
（3）新增/编辑/删除成功或失败时，显示对应 ElMessage 提示。
（4）导出 CSV 成功后，浏览器下载 CSV 文件。
（5）行点击后弹出 el-dialog 详情窗口，展示设备全部字段。

**4、实现方式**

进行设备列表查询操作时，进入 DeviceList.vue 页面。onMounted 触发 fetchDevices() 方法，该方法构建查询参数（page、pageSize、name、status），调用 api/device.js 中的 getDevices() 函数发送 GET /api/devices 请求。后端路由经 authMiddleware 验证 JWT、requirePermission 校验 device:read 权限后，由 deviceController.getDevices() 调用 Device.findAll() 模型方法。该方法使用参数化查询拼接 SQL（含 LIKE 模糊搜索（通配符已转义）、IN 多状态筛选、LIMIT 分页），返回分页数据。前端收到响应后更新 devices.value 和 total.value，el-table 自动渲染。

进行设备新增操作时，进入 DeviceForm.vue 页面。用户填写表单后点击提交按钮，触发 handleSubmit() 方法。该方法先执行 formRef.validate() 进行前端验证（必填项、字段长度），验证通过后调用 createDevice() 发送 POST /api/devices 请求。后端同样执行输入校验（必填、状态枚举、字段长度、日期格式），校验通过后 Device.create() 执行 INSERT SQL 并返回完整记录。

进行 CSV 导出时，handleExport() 方法调用 exportCSV() 工具函数，将当前页 devices.value 数据按 CSV 格式拼接（含 BOM 头确保 UTF-8 编码、公式注入防护），创建 Blob 对象并通过临时 <a> 元素触发下载。

进行排序操作时，用户点击 el-table-column 的排序按钮，触发 @sort-change 事件调用 handleSortChange()，更新 sortProp 和 sortOrder 状态。sortedDevices 计算属性根据排序字段和顺序对 devices.value 进行客户端排序，el-table 的 :data 绑定 sortedDevices 实现排序渲染。

**5、流程图**

```
用户进入设备列表
      ↓
  onMounted → fetchDevices()
      ↓
 GET /api/devices?page=1&pageSize=20
      ↓
 authMiddleware(验证JWT) → requirePermission(device:read)
      ↓
 Device.findAll() → 参数化SQL → MySQL
      ↓
 返回 { list, total, page, pageSize }
      ↓
 渲染 el-table + el-pagination
      ↓
 用户操作:
  ├── 筛选/翻页 → onFilterChange → 重置page=1 → fetchDevices
  ├── 排序 → @sort-change → sortedDevices 计算属性重排
  ├── 点击行 → getDevice(id) → 显示详情弹窗
  ├── 编辑 → goEdit(id) → DeviceForm.vue(编辑模式)
  ├── 删除 → popconfirm确认 → deleteDevice(id) → 刷新列表
  ├── 新增 → goAdd → DeviceForm.vue(新增模式)
  └── 导出 → exportCSV() → 下载CSV
```

#### 3.1.2 用户与权限管理功能

**1、功能描述**

用户管理用于管理员对系统用户的管理，支持查看用户列表（含权限详情）、新增用户、删除用户（禁止删除自己和管理员账号）、分配设备操作权限（实时生效，事务保护）。

认证功能支持用户注册和登录。登录成功后返回 JWT token，前端持久化到 localStorage。路由守卫在每次页面跳转时检查 token 存在性，未认证跳转登录页。

**2、输入项目**

（1）管理员新增用户：输入用户名（3-50字符）、密码（≥6字符）、昵称（可选）。
（2）管理员分配权限：勾选/取消设备操作的四个权限 checkbox。
（3）管理员删除用户：点击删除按钮并确认。
（4）用户注册：输入用户名、密码、昵称。
（5）用户登录：输入用户名、密码。

**3、输出项目**

（1）用户列表展示：以表格展示所有用户的 ID、用户名、昵称、角色、权限状态。
（2）新增用户成功/失败提示。
（3）删除用户成功/失败提示（后端拒绝时显示具体原因）。
（4）权限更新成功提示。
（5）登录成功返回 JWT 并跳转；登录失败显示错误提示。

**4、实现方式**

进行用户管理操作时，进入 UserManage.vue 页面（仅 admin 可访问，由路由守卫和 adminOnly 中间件双层保护）。onMounted 触发 fetchUsers() 调用 getUsers() 发送 GET /api/users 请求。后端 permissionController.listUsers() 先查询全部用户，再通过 Permission.getPermissionsByUsers() 批量查询权限（一次 IN 查询避免 N+1），合并后返回。前端将权限数据存入 permsMap，el-checkbox-group 通过 :model-value="getPerms(row.id)" 绑定权限值。

权限 checkbox 变更时触发 onPermChange()，调用 setUserPermissions() 发送 PUT /api/users/:id/permissions 请求。后端 Permission.bulkSetPermissions() 使用事务（BEGIN → DELETE → INSERT → COMMIT/ROLLBACK）确保原子性。

用户认证由 Login.vue 页面实现。登录时调用 login() 发送 POST /api/auth/login 请求，后端 authController.login() 使用 bcrypt.compare() 校验密码，通过后 jwt.sign() 签发 token。前端将 token 和 userInfo 分别存入 localStorage，路由守卫 router.beforeEach 在每次导航时检查 token 存在性。

#### 3.1.3 数据统计与生命周期配置功能

**1、功能描述**

Dashboard 数据概览页展示设备总数、各状态设备数量（正常运行/维保中/已报废）、即将维保数量和最近添加的 5 台设备列表。每次访问 Dashboard 时自动触发设备的生命周期状态流转：正常设备超过 N 个月未维保自动变为"维保中"；维保中设备超过 M 个月自动变为"已报废"。每台设备的 N、M 值在新增/编辑设备时配置，默认分别为 11 和 12。

**2、输入项目**

（1）用户访问 Dashboard：无输入，自动加载统计数据。
（2）管理员配置生命周期：在设备表单中设置维保触发月数和报废触发月数。

**3、输出项目**

（1）展示 5 个统计卡片：设备总数（蓝色）、正常运行（绿色）、维保中（橙色）、已报废（灰色）、即将维保（红色）。
（2）展示最近添加的 5 台设备表格。
（3）展示快捷操作按钮（新增设备、设备列表、用户管理（admin））。

**4、实现方式**

Dashboard.vue 页面在 onMounted 时调用 fetchStats() 发送 GET /api/devices/stats 请求。后端 deviceController.getDeviceStats() 调用 Device.getStats()，该方法首先执行 Device.applyLifecycleRules() 进行状态流转（使用每台设备自身阈值的 UPDATE SQL），然后依次执行 3 条统计查询：① GROUP BY status 统计各状态数量；② TIMESTAMPDIFF 计算即将维保数量；③ LIMIT 5 查询最近设备。前端将返回数据渲染为统计卡片和表格。

生命周期配置在 DeviceForm.vue 中实现。表单底部 el-input-number 绑定 maintenance_interval 和 scrap_interval，默认值分别为 11 和 12。保存时与设备其他字段一起提交，Device.create()/update() 的 INSERT/UPDATE SQL 包含这两个字段。
